## 1. 测试策略

### 1.1 测试金字塔

```
         ┌──────────┐
         │  E2E 测试  │  少量 — 完整用户流程
        ┌┴──────────┴┐
        │  集成测试   │  中量 — 模块间交互
       ┌┴────────────┴┐
       │   单元测试      │  大量 — 函数/类级别
       └────────────────┘
```

| 层级 | 占比目标 | 范围 | 框架 |
|------|----------|------|------|
| 单元测试 | 70% | 单个函数/类，mock 外部依赖 | Python: pytest / Go: go test / Rust: cargo test |
| 集成测试 | 20% | 模块间交互（Python↔Rust FFI、Go↔Python gRPC） | pytest + grpcio / go test |
| E2E 测试 | 10% | 完整用户流程（发消息→模型响应→工具执行→权限确认） | pytest + subprocess |

### 1.2 各层测试范围

| 层 | 语言 | 测试重点 |
|----|------|----------|
| Python 上层 | pytest | Agent 循环逻辑、模式切换、工具调度、权限判断、记忆 CRUD、模型路由、上下文压缩、错误恢复 |
| Go 中间层 | go test | API 端点、SSE 流式、认证、限流、gRPC 转发 |
| Rust 底层 | cargo test | Token 计数准确性、工具并行执行、Docker 容器生命周期、FTS5 索引一致性 |

## 4. E2E 测试

### 4.1 完整用户流程

| 用例 | 步骤 | 预期 | Phase |
|------|------|------|-------|
| **基础对话** | 用户发送消息 → Agent 调用模型 → 流式返回文本 | 正确的 SSE 事件流，done 事件包含正确统计 | Phase 1 |
| **工具调用（read_file）** | 用户要求读文件 → Agent 调用 read_file → 返回内容 | 权限检查 → 工具执行 → 结果返回 → 模型继续生成 | Phase 1 |
| **工具调用（terminal）** | 用户要求执行命令 → Agent 调用 terminal → ask-every 权限确认 → 用户确认 → 执行 | SSE 中出现 permission_request 事件，确认后继续 | Phase 1 |
| **模式切换** | 用户发送代码任务 → 自动检测建议 → 用户确认 → 切换 Coding 模式 → 工具集变化 | mode_suggest 事件 → 确认后只加载 coding 工具 | Phase 1 |
| **上下文压缩** | 长对话超阈值 → 触发 auto-compact → 模型压缩 → 继续对话 | 压缩后对话继续正常 | Phase 1 |
| **错误恢复** | 模型超时 → fallback 到备用模型 → 继续对话 | 自动切换，用户感知到延迟但结果正确 | Phase 1 |
| **记忆持久化** | 对话中用户说偏好 → Agent 写入记忆 → 下次对话注入 | 记忆出现在后续对话的 system prompt 中 | Phase 1 |
| **沙箱执行** | Coding 模式下运行代码 → Docker 容器创建 → 执行 → 销毁 | 容器生命周期正确，输出正确，容器不残留 | Phase 1 |

### 4.2 E2E 测试实现方式

```python
# test_e2e.py
# 通过 subprocess 启动完整的 Khaos 进程（Go 网关 + Python Agent + Rust FFI）
# 模拟用户通过 CLI 发送消息，验证 SSE 事件流

import subprocess
import pytest
import asyncio


@pytest.fixture(scope="session")
async def khaos_server():
    """启动完整的 Khaos 服务"""
    proc = subprocess.Popen(
        ["go", "run", "./cmd/gateway"],
        stdout=subprocess.PIPE,
        stderr=subprocess.PIPE,
    )
    # 等待健康检查通过
    await wait_for_health("http://localhost:8080/api/health")
    yield proc
    proc.terminate()


async def test_basic_chat(khaos_server):
    """E2E: 基础对话"""
    response = await send_chat("你好")
    events = parse_sse(response)
    assert events[-1].type == "done"
    assert events[-1].data["total_tokens"] > 0


async def test_tool_call_with_permission(khaos_server):
    """E2E: 工具调用 + 权限确认"""
    response = await send_chat("读取 /tmp/test.py 的内容")
    events = parse_sse(response)
    # 应该有 permission_request 事件
    perm_events = [e for e in events if e.type == "permission_request"]
    assert len(perm_events) == 1
    # 确认权限
    await confirm_permission(perm_events[0].data["tool_call_id"], approved=True)
    # 继续接收后续事件
    remaining = await continue_stream(response)
    done = [e for e in remaining if e.type == "done"]
    assert len(done) == 1
```

| 并发子代理 | ≥ 3 个同时运行 | Phase 2。同时 spawn 3 个子代理全部正常完成 |
| gRPC 通信延迟 | < 10ms（本地） | Go ↔ Python 单次 RPC 往返 |

## 6. 安全测试

跨平台安全测试按 `docs/platform-security-ci.md` 分为 contract matrix、OS real sandbox、
Docker real sandbox。Ubuntu、Windows、macOS 使用 GitHub-hosted runner；能力缺失必须失败
或明确 Unsupported，不能回退 Host。所有第三方 Action 固定 commit SHA，workflow token
保持 `contents: read`。

| 测试项 | 方法 | 预期 | Phase |
|--------|------|------|-------|
| API Key 泄露 | 检查日志中是否出现明文 API Key | 全部脱敏为 `***` | Phase 1 |
| 路径穿越 | 工具参数传入 `../../../etc/passwd` | 权限规则拦截或路径规范化 | Phase 1 |
| 命令注入 | terminal 工具传入 `; rm -rf /` | 命令分段安全评估拦截 | Phase 1 |
| Docker 逃逸 | 沙箱内尝试访问宿主机 | 网络隔离 + 权限限制阻止 | Phase 1 |
| SQL 注入 | 记忆 key/value 传入 SQL | 参数化查询，无注入风险 | Phase 1 |
| 权限绕过 | 未授权操作直接调用 API | 权限引擎拦截，返回 deny | Phase 1 |
| UDS 未认证或重放 | 缺少 boot capability、伪造 principal、修改 method/payload、复用 nonce | 全部拒绝；只有匹配 peer credential 且签名有效的 Gateway 请求可 dispatch | Phase 4 |
| Task 审批竞态 | 并发审批、陈旧 capability、CAS 失败、BLOCKED task 直接恢复 | capability 必须先消费；任务从不短暂暴露 RUNNING；消费证据持久化 | Phase 4 |

## 9. CI 集成

### 9.1 CI 流水线

```yaml
# .github/workflows/test.yml（Phase 2 开源时启用）

on: [push, pull_request]

jobs:
  python-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r python/requirements.txt
      - run: pip install pytest pytest-cov
      - run: pytest python/tests/ --cov=khaos --cov-report=xml

  go-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: "1.22" }
      - run: go test -cover ./go/...

  rust-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test && cargo tarpaulin --out Xml
    services:
      docker:
        image: docker:dind

  integration-test:
    needs: [python-test, go-test, rust-test]
    runs-on: ubuntu-latest
    steps:
      - run: pytest python/tests/integration/
    services:
      docker:
        image: docker:dind
```
