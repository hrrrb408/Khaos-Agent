# ADR-009：统一 ExecutionBackend 与安全拒绝

## 状态

已接受（Phase 0）。

## 决策

终端、测试和沙箱工具统一通过 ExecutionBackend。后端不可用时，`workspace-write` 请求拒绝执行，不回退到裸 `subprocess`；默认网络策略为 `none`。

## 原因

逻辑权限、Worktree 边界、OS/Docker 隔离、资源限制和审计必须形成纵深防御。

## 替代方案

- 每个工具自行启动进程：安全策略容易漂移。
- 后端不可用时裸执行：会把“无法隔离”错误地变成“允许写入”。

## 回滚与兼容

保留旧工具入口，通过适配器委托。只读操作可在兼容模式运行；可写操作必须显式选择安全后端或得到结构化拒绝。
