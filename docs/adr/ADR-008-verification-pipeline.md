# ADR-008：VerificationPipeline 作为分层验证入口

## 状态

已接受（Phase 0）。

## 决策

项目探测、验证计划、执行、解析、受影响测试和报告由 VerificationPipeline 组合；现有 `test_run` 保留兼容入口并委托该服务。

## 原因

单一测试命令无法表达 lint、类型检查、构建、测试、超时、flaky 和基线差异。结构化步骤也便于审计和 Verify-Fix 消费。

## 替代方案

- 继续把命令字符串交给 Agent：不可稳定审计，也无法统一超时与资源策略。
- 为每个生态复制一套执行循环：重复逻辑多，难以保证安全策略一致。

## 回滚与兼容

显式 `test_run(command, cwd)` 映射为单步计划；旧结果字段保留，新字段以可选方式增加。
