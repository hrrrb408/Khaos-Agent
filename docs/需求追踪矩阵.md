## Phase 1 子阶段划分（P0-A / P0-B / P0-C）

### P0-A：跑通最小闭环

> 交付物：CLI 能发消息，mock 模型返回固定文本，SSE 流式输出

| 编号 | 功能 | HLD 模块 |
|------|------|----------|
| FR-001 | Agent 核心循环（mock 模型，固定响应） | §2.1 Agent 核心 |
| FR-002 | 双模式切换（命令切换，工具集切换） | §2.2 模式管理 |
| FR-015 | CLI 界面（基础，能发消息能收到响应） | §3.3 平台接入（CLI） |
| — | 数据库 Schema（全部表） | §5 数据层设计 |
| — | 单 Provider router skeleton | §2.8 模型路由 |

### P0-B：工具闭环

> 交付物：Agent 能调用真实工具，权限确认流程跑通

| 编号 | 功能 | HLD 模块 |
|------|------|----------|
| FR-005 | 工具注册与调度 | §2.3 工具系统 |
| FR-006 | 文件操作工具 | §2.3 工具系统（文件子集） |
| FR-007 | 终端工具 | §2.3 工具系统（终端子集） |
| FR-013 | 权限引擎（ask-every + 单次确认 + remember rule） | §2.5 权限引擎 |
| — | SSE tool_result / permission_request 事件 | §2.2 SSE 消息格式 |

### P0-C：工程增强

> 交付物：完整 MVP，所有 P0 功能可用

| 编号 | 功能 | HLD 模块 |
|------|------|----------|
| FR-003 | 上下文压缩（三层：micro-compact、context-collapse、auto-compact） | §2.7 上下文压缩 |
| FR-008 | Docker 沙箱（sandbox_exec、sandbox_build） | §2.3 工具系统（沙箱子集） |
| FR-014 | 记忆系统（SQLite CRUD + FTS5 搜索 + 基础注入） | §2.4 记忆系统 |
| FR-017 | 错误处理与恢复（fallback、熔断） | §2.1 Agent 核心（错误处理内嵌于循环） |
