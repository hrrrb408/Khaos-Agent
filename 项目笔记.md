## 命名故事

### 为什么叫 Khaos

1. 瑞邦的个人ID是「绯鸢」——鸢是猛禽，自己捕猎自己判断，不是等着喂的。绯是暗红色，压着火的那种。
2. 产品名从绯鸢衍生：绯→炼，「炼鸢」——把六种材料炼成一只鸢。后来改了方向。
3. 最终定了「混沌」——瑞邦这个人最核心的特质就是混沌。不是贬义，天地未分的状态，什么都有可能。
4. 英文名 Khaos 取希腊词根，跟主流产品不撞，视觉上够狠。

**文档编写过程中的教训：**

- 章节重排用全局正则替换会交叉污染子节编号，必须按章节边界切片后针对性替换
- Python 脚本重写文件容易引入重复内容和顺序错乱，关键操作要做验证
- ch15（集成建议）内容在重排过程中丢失，需要从上下文重新补写

## 竞品调研核心发现

### 六家各自最值得学的

| 竞品 | 最值得学 | 具体是什么 |
|------|----------|-----------|
| Claude Code | 核心循环 | while(true) + 流式API + 7层压缩 + 工具并行/串行 + 三层协作 |
| Claude Code | 上下文压缩 | micro-compact → context collapse → auto-compact → session memory → auto dream |
| Claude Code | 权限设计 | 规则引擎 allow/deny/ask + glob匹配 + 规则持久化 |
| Codex | 沙箱 | 跨平台沙箱（Seatbelt/Bubblewrap/ACL/WFP）+ 命令分段安全评估 |
| Codex | 权限三维交叉 | 审批×沙箱×协作 = 48种理论组合 |
| Codex | 权限持久化 | prefix_rule 命令前缀规则 |
| OpenClaw | 工具策略管道 | 7层过滤（签名/解密/验证/限流/审计/路由/序列化） |
| OpenClaw | Fallback 系统 | 模型不可用时自动切换 |
| Hermes | 记忆系统 | 三层记忆 + 五维优化 + 跨模式同步 |
| Hermes | 子代理 | delegate_task 平级委托 + 最大并发控制 |
| Hermes | Session Search | FTS5 搜索历史会话 |
| Marvis | 四级调度 | Main → Sub → Skill → Tool |
| Marvis | ask_user | 结构化确认卡片（六家唯一） |
| Marvis | 记忆精细度 | 对话记忆表 + 长期记忆表 + FTS5段落级检索 + 审计日志 + 用户偏好表 |
| Marvis | WriteAudit | 所有写操作可追溯 |

### 六家共同缺失

| 缺失 | 说明 |
|------|------|
| 双模式切换 | 没有任何一家实现了办公/Coding 运行时切换 |
| 子代理嵌套 | Hermes 不嵌套，Claude Code 三层不是嵌套 spawn |
| 按功能路由模型 | 大多是单模型或简单 fallback |
| 开源+闭源竞品对比 | 没有 agent 做过这类全面分析 |
